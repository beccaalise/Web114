Questions & Answers

1. Why is the JavaScript textContent property safer than innerHTML?

The textContent property is safe than innerHTML because it only handles plain text and does not treat content as HTML. As a result, it prevents the execution of embedded scripts, reducing the risk of cross-site scripting attacks. Whereas innerHTML interprets and renders HTML content, which can pose security threats when dealing with untrusted user input.

2. Explain the relationship between an HTML attribute and a corresponding DOM property. Use a specific example.

HTML attributes are the starting values you set in the HTML code, while DOM properties are what JavaScript uses to work with those elements on the page. When a webpage loads, the browser turns attributes into properties so you can interact with them using JavaScript. For example, <input id="myInput" value="Hello"> the value attributes sets the initial text.
